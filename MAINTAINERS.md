# Maintainers

This file records who is responsible for ongoing project stewardship.

| Maintainer | Role | Responsibilities |
| --- | --- | --- |
| `niguanglang` | Owner / primary maintainer | Repository stewardship, PR review, issue triage, release planning, security-sensitive review, and OpenAI Codex for OSS application ownership. |

## Maintainer Duties

Maintainers should:

- Review incoming pull requests and require passing CI before merge.
- Triage new issues with severity, affected area, reproduction quality, and
  release impact.
- Keep security-sensitive code paths covered by tests and documentation.
- Maintain release notes, upgrade notes, and production deployment guidance.
- Close stale or non-actionable issues with a short explanation.
- Coordinate private vulnerability reports according to [SECURITY.md](./SECURITY.md).

## Becoming a Maintainer

Regular contributors may be invited as maintainers after sustained,
high-quality contributions that show judgment in one or more areas: Web
Console, Control API, Agent Runtime, deployment, documentation, tests, or
security review.
