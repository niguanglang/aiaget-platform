# M144 审批导出筛选上下文前端提示

## 背景

M143 已经让统一安全审批工作台 CSV 导出包含通知归档筛选来源、筛选状态和筛选关键词。但 `/security/alerts` 页面导出区域仍只提示“导出会写入安全审计事件”，用户在导出前无法从界面确认 CSV 会带上通知归档筛选上下文。

## 范围

本里程碑只调整前端导出提示：

1. `/security/alerts` 统一审批工作台导出说明提示“导出会包含通知归档筛选上下文”。
2. 导出成功提示说明 CSV 已包含“通知筛选来源、状态和关键词”。
3. 不新增导出按钮、不新增 CSV 预览、不改变审批列表字段。
4. 不改变后端导出契约、表结构或中间件。

## 前端设计

页面仍使用现有统一审批工作台区域：

```text
当前筛选命中 N 条，导出会写入安全审计事件，并且导出会包含通知归档筛选上下文。
```

导出成功后显示：

```text
审批工作台导出完成，当前筛选命中 N 条，CSV 已包含通知筛选来源、状态和关键词。
```

文案只作为导出结果说明，不把通知审计明细、客户成功报告正文或归档删除详情加入 `/security/alerts` 列表。

## 验收标准

1. 前端 IA 契约包含“导出会包含通知归档筛选上下文”。
2. 前端 IA 契约包含“通知筛选来源、状态和关键词”。
3. `/security/alerts` 仍只保留原导出按钮和筛选区域。
4. 前端定向 IA 测试、Web typecheck、Web IA 全量测试通过。
