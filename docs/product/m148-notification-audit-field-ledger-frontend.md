# M148 通知审计字段账本前端提示

## 背景

M147 已经把审批工作台导出字段账本写入运营告警通知审计 payload，并让通知审计 CSV 包含“导出字段清单”和“通知归档筛选字段”。前端 `/security/alerts` 的通知审计卡片还只提示筛选条件会被导出，没有说明字段账本已经进入 CSV，也没有给审计员轻量识别哪些通知带有字段账本。

## 范围

本里程碑只调整告警运营页的通知审计区域：

1. 在通知审计卡片顶部增加“通知审计字段账本”提示。
2. 导出成功反馈说明 CSV 已包含导出字段清单和通知归档筛选字段。
3. 通知审计列表行只显示轻量“字段账本 数量/数量”标签。
4. 不在列表中展开完整字段数组。
5. 不新增接口、不新增页面、不改变后端契约。

## 前端设计

列表行保持核心字段：

```text
状态 / 来源 / 字段账本 5 / 3 / 告警短 ID
消息摘要
重试次数 / 创建时间
```

完整字段范围仍由 CSV 和安全事件详情承担，列表只作为运营扫描入口。

## 验收标准

1. `/security/alerts` 源码包含“通知审计字段账本”。
2. 导出成功提示包含“CSV 已包含导出字段清单和通知归档筛选字段”。
3. 列表读取 `exported_fields` 和 `notification_archive_filter_fields`。
4. 列表不渲染通知审计 item 的 JSON 详情。
5. 安全中心 IA 测试、Web typecheck 和 Web IA 全量测试通过。
