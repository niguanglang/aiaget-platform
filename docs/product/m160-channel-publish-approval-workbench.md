# M160 渠道发布审批接入统一安全审批工作台

## 背景

渠道发布控制已经具备本地审批状态机，支持发起、通过、拒绝、灰度、全量和回滚。但待审批事项仍停留在渠道模块内部，安全中心统一审批工作台无法统一查看和处理渠道发布审批。

## 本次收敛

1. 新增统一审批类型 `CHANNEL_PUBLISH_APPROVAL`。
2. 新增审批风险域 `CHANNEL`，用于区分渠道发布风险与工具、策略、审计归档和运营告警风险。
3. 统一安全审批工作台加载开启发布审批且状态为 `PENDING`、`APPROVED`、`REJECTED` 的渠道。
4. 工作台详情展示渠道类型、关联 Agent、灰度比例、回滚可用性、渠道状态和健康状态等扩展信息。
5. 工作台审批动作转发到现有 `ChannelsService.approvePublish` 与 `ChannelsService.rejectPublish`，不重写渠道发布状态机。
6. 前端安全中心“告警运营 / 审批工作台”筛选项增加“渠道发布审批”和“渠道发布”风险域。

## 权限语义

- 原渠道发布审批接口继续保留 `channel:publish:deploy` / `channel:publish:disable` 权限和 `CHANNEL` 数据权限、资源 ACL 边界。
- 统一审批工作台入口继续使用 `security:approval:view` 和 `security:approval:handle`。
- 工作台只是统一聚合和转发，最终审批仍落到渠道发布控制的既有服务逻辑。

## 影响范围

本次不改变渠道发布配置结构、不新增数据库表、不改灰度发布和回滚流程。安全中心会将渠道发布审批纳入统一待办统计、筛选、详情和处理链路。
