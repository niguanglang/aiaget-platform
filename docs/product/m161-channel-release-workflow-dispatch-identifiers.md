# M161 渠道发布工作流派发标识贯通

## 目标

渠道发布自动推进和发布自愈在派发 Runtime 工作流后，需要立即把工作流后端、工作流 ID、运行 ID 返回给控制面和调度器，方便运维侧在 Temporal / Runtime 链路中快速定位异步任务。

## 本次完成

- 渠道自动推进工作流派发结果新增 `workflow_id`、`workflow_run_id`。
- 渠道发布自愈工作流派发结果新增 `workflow_id`、`workflow_run_id`。
- 发布巡检调度器的渠道级结果和完成事件 payload 保留工作流标识。
- 自动推进 / 自愈运行结果持久化时保留 `workflow_run_id`，历史配置归一化读取时同步返回。
- 本地执行和 Temporal fallback 路径保持字段结构一致，无法产生 Runtime 标识时返回 `null`。

## 影响范围

- Control API：渠道发布工作流服务、发布巡检调度器、渠道运行结果归一化。
- Shared Types：渠道自动推进、自愈、调度结果类型。
- Runtime：不需要改动，沿用已有 `workflow_id`、`run_id` 响应。

## 约束

- 未新增数据库表结构。
- 未启动或新增任何中间件、容器。
- 未改变已有审批、安全策略和发布执行逻辑，只补齐异步工作流可观测标识。
