# M63-21 渠道发布事故复盘与变更报告中心

## 目标

把渠道发布过程中的批次、审批、灰度、自动推进、自愈、巡检、事件和用量信号汇总成一份可审计的中文复盘报告。

本模块不新增数据库表，报告按需从现有数据生成。

## 新增接口

```text
GET /channels/:channelId/release-report
```

权限：

```text
channel:publish:view
```

## 数据来源

```text
agent_publish_channel.config
platform_event
platform_usage_event
```

覆盖信息：

```text
1. 当前发布批次
2. 审批状态
3. 灰度状态
4. 回滚点
5. 自动推进最近结果
6. 发布自愈最近结果
7. 发布巡检事件
8. 近 72 小时失败事件
9. 近 72 小时用量失败信号
10. Trace / Request ID 线索
```

## 前端

在 `/channels` 页面新增“渠道发布复盘与变更报告”面板：

- 复盘结论
- 风险等级
- 核心指标
- 风险与建议
- 关键时间线
- Markdown 报告正文

## 边界

- 不持久化报告快照。
- 不做导出文件。
- 不启动容器。
- 不安装中间件。
- 不执行数据库迁移。

后续如果需要合规归档，可以新增报告快照表，并为每次发布批次生成不可变更报告版本。

## 验收标准

- 能按渠道生成中文复盘报告。
- 报告包含风险、建议和时间线。
- 前端可查看指标、风险卡片、时间线和 Markdown 正文。
- Control API 和 Web typecheck 通过。
