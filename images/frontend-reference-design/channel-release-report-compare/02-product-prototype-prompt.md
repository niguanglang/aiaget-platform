# Product Prototype / Wireframe Prompt

绘制 `/channels/release/reports` 渠道发布复盘报告页的中保真线框原型，中文标注，聚焦信息架构和交互流。

页面结构：
1. 页面头部：标题“发布复盘报告”、说明、刷新按钮。
2. 渠道选择器：卡片式或分段列表，用于切换渠道。
3. 报告摘要区：6 个核心字段 + “创建快照”操作。创建快照需要二次确认。
4. 风险建议区：空态或风险条目列表。
5. 复盘快照区：快照概览列表。每条包含“查看快照”“设为基准”“设为对比”三个行内按钮。
6. 快照详情面板：输入或选择快照 ID 后加载详情，展示基础字段、归档报告正文、来源事件。
7. 版本对比面板：显示基准/对比快照 ID、开始对比按钮、自动选前两项按钮、对比摘要和四类差异列表。

原型重点：
- 列表页不展示完整详情，完整报告正文只在摘要和详情面板出现。
- 快照列表只保留识别字段、状态字段和行内操作。
- 对比结果必须有清晰的 before/after 对照。
- 错误、空态、加载、禁用、成功提示都要预留位置。
