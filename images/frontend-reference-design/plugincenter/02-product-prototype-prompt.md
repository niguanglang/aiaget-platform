# Product Prototype / Wireframe Prompt

生成一张中保真产品原型图，中文界面，桌面端 16:10，页面为企业 Agent 平台的「插件生态中心」。

原型重点是信息架构和交互流程，不要过度视觉装饰：
- 页面顶部：标题「插件生态中心」，右侧操作「刷新」「查看市场」。
- 第一行：4 个统计块，分别是「插件总数」「已启用」「待审核」「菜单注入」。
- 第二行左右两栏：
  - 左栏宽度约 58%，是「插件列表」卡片。
  - 右栏宽度约 42%，是「插件详情」卡片。
- 插件列表卡片：
  - 顶部说明文字。
  - 分段控件：市场、已安装。
  - 搜索框、状态下拉、风险下拉、清空按钮。
  - 表格列：插件、版本、状态、风险、能力、操作。
  - 空状态：标题「暂无插件」，描述根据市场/已安装切换。
- 插件详情卡片：
  - 未选择时显示 Empty State：「选择插件查看详情」。
  - 已选择时显示：状态徽标、名称、编码、提供方、来源、配置/启停/升级按钮。
  - 下面是三列基础信息：已安装版本、最新版本、安装时间。
  - 安全预览区：摘要 + 风险说明。
  - 两个并列区域：菜单注入、Hook 绑定。
  - 两个并列区域：权限预览、审计记录。
- 编辑抽屉：
  - 从右侧滑出，标题「编辑插件配置」。
  - 字段：名称、最新版本、安装状态、运行状态、风险等级、描述、配置 JSON。
  - 底部操作：取消、保存修改。

权限和状态表达：
- 无 `plugin:center:view` 时显示无权限 Empty State。
- 没有 manage/install/enable/disable/upgrade 权限时对应按钮禁用。
- Loading、Error、Success Notice、Form Validation Error 都要有占位。

视觉限制：
- 线框以灰度为主，少量蓝色表示主按钮和选中态。
- 不要营销页结构，不要大 Hero，不要装饰性大插画。
- 保持三层以内信息层级，确保后续能用 Tailwind + shadcn/ui 风格组件直接实现。
